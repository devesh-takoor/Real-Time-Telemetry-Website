How it works?





How we actually collect data?


First of all we connect our Arduino Board(Uno Wifi Rev2 - You can find yours here.) and then we upload our code.

You must change your topics from the arduino code to avoid overriding your data and also my data.

The data will then be sent to a MQTT-Broker



Do not forget to put the Wifi_SSID and Wifi_Password in the "arduino_secrets.h" file of the place where your Arduino Board is.

Now let us see how to store the data!

How to store the data?


First of all we must retrieve the data from the MQTT-Broker using all the information needed(broker name, broker port, broker topic)

Then we copy this code, of course you must put your topic according to what you have written previously on the Arduino IDE.

Run the code, so that it will receive the data from the MQTT-Broker



It will be automatically be updated in the Database as per the code of the python file

Display the Data


To display the data you must log in to the page which you will see on the navbar at the top.

Once logged in you go to the Dashboard page where you will be able to see all your data from your sensors on the graph



The graph will be updated every 10 minutes as per the code, also 10 minutes for not overloading the database quickly (it all depends on the user).
